# Ubiquitous Language

This document defines the core vocabulary used throughout TranscribeBench.
These terms should be used consistently in:

- folder names
- CLI commands
- documentation
- configuration files
- architecture descriptions

If new concepts are introduced to the project, they should be added here.

## Benchmark
A benchmark definition that specifies which engine/model pairs, dataset subset, and parameters are used for evaluation.

## Benchmark run
A single execution of a benchmark that produces results and reports.

## Dataset
A source collection of audio recordings and reference transcripts.

## Dataset subset
A fixed selection of samples from a dataset used for a benchmark run.

## Engine
A transcription runtime adapter (for example `mlx_whisper`, `whisper_cpp`, `parakeet_mlx`).

## Model
Neural network weights loaded by an engine (for example `whisper-large-v3`).

## Engine/model pair
The concrete runtime + model combination evaluated in a benchmark run.

## Configuration
A YAML file describing benchmark parameters (dataset settings, engine/model pairs, output paths).

## Results
Machine-readable output generated by a benchmark run (JSON).

## Benchmark report
Human-readable summary of benchmark results (Markdown / CSV).

## Cache
Rebuildable downloaded or generated data such as dataset subsets or model files.

## Prepare environment
Actions that make runtimes available locally (for example dependency checks or building whisper.cpp).

## Refresh dataset
Rebuild or repopulate the dataset cache so it matches the current configuration.

## Avoid (non-preferred terms)

These terms appeared in earlier versions of the project but should be avoided in new documentation or interfaces.

- `bench` -> use **benchmark**
- `fetch` -> use **refresh dataset** or automatic dataset preparation
- `setup` -> use **prepare environment**
- `runs` (as a concept) -> use **benchmark runs**
- `report output` -> use **benchmark report**
